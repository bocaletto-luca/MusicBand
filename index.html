StudioBand

Terminal-Based Audio & Music Production App Manager for Debian & Ubuntu

Get Started

Features

  • Pre-configured catalog of popular audio/music apps (DAWs, synths, players, JACK/PipeWire tools).
  • Instant status overview: ✔️ installed vs. ✖️ not installed.
  • One-step install or remove; choose between remove (keep config) or purge (wipe config).
  • Launch multiple installed apps in parallel from a checklist menu.
  • Extendable “custom apps” system—add any APT package + launch command.
  • Persistent settings under ~/.studioband/ for future sessions.

Requirements

  • Debian or Ubuntu (and derivatives).
  • Bash 4.x or newer.
  • whiptail (install with sudo apt-get install -y whiptail).
  • Sudo privileges.

Installation

# 1. Clone the repository
git clone https://github.com/bocaletto-luca/StudioBand.git
cd StudioBand

# 2. Make the script executable
chmod +x StudioBand.sh

# 3. Install whiptail (if needed)
sudo apt-get update
sudo apt-get install -y whiptail

# 4. Launch StudioBand
./StudioBand.sh

Usage

After launching ./StudioBand.sh, you’ll see a main menu with the following options:

  1. Install / Remove applications: Select apps to install (OFF) or remove (ON). If removing, choose remove (preserve config) or purge (delete config).
  2. Show application status: View a ✔️/✖️ list of installed vs. not installed.
  3. Launch applications: Select one or more installed apps to start in the background.
  4. Add custom app: Define an ID, descriptive name, APT package and launch command. Entries are saved to ~/.studioband/apps.custom.
  5. Info / Help: Displays version, config file location and credits.
  6. Exit: Close StudioBand cleanly.

Adding Custom Apps

To include an application not in the built-in list:

  1. Select Add custom app from the main menu.
  2. Enter a unique ID (e.g., mydrum), a descriptive name, the APT package and the launch command.
  3. StudioBand appends your entry to ~/.studioband/apps.custom and reloads—your new app appears immediately.

Download the latest script directly: StudioBand.sh (raw)

Customization

You can tailor StudioBand by editing the built-in app arrays at the top of StudioBand.sh:

  • APP_NAME – Human-readable labels.
  • APP_CMD – Commands to launch each app.

Your custom entries are stored in ~/.studioband/apps.custom. Back up or restore your setup by copying the entire ~/.studioband/ folder.
